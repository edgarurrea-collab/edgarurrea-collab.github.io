Home

Este sitio web estático recopila las evidencias conceptuales y de desarrollo del proyecto práctico integrador para el curso de Sistemas Operativos. El propósito de esta entrega consiste en construir y documentar detalladamente una infraestructura de red que combine virtualización clásica, aislamiento de servicios mediante contenedores Docker y orquestación resiliente en clústeres locales usando Kubernetes.

Especificaciones Asignadas al Grupo 7:
  • Entorno de Escritorio Virtual (Gráfico): Zorin OS 17
  • Entorno de Servidor Virtual (Consola): Ubuntu 24.04 Server
  • Imagen de Contenedor Base: debian:13

Componentes del Proyecto

A continuación se detallan las actividades obligatorias realizadas para cada fase del laboratorio, incluyendo los comandos fundamentales usados en consola y la visualización de la infraestructura.

Componente 1: Virtualización con Linux (30%)

Se configuró un entorno virtualizado local compuesto por dos Máquinas Virtuales interconectadas de la siguiente manera:

  • VM 1 (Zorin OS 17): Interfaz gráfica completa para administración del clúster y desarrollo.
  • VM 2 (Ubuntu 24.04 Server): Servidor puro por terminal CLI (sin entorno gráfico).
  • Red e Integración: Esquema de particionamiento manual independiente (/, swap, /home) en cada disco, red configurada mediante adaptador puente/red interna y comunicación segura transparente vía SSH desde la máquina gráfica a la consola.

Comando esencial de verificación SSH:

ssh usuario_servidor@192.168.100.X
[Aquí debes incluir tus capturas: Instalación, Particionamiento Manual, Configuración de Red y Prueba de Conexión SSH]

Componente 2: Contenedores Docker (30%)

En la distribución gráfica (Zorin OS 17) se instaló directamente Docker Engine (comunidad nativa) y se estructuró una aplicación multi-contenedor orquestada mediante Docker Compose basada en la imagen limpia asignada de debian:13.

  • Frontend: Servidor web Nginx sirviendo archivos HTML estáticos respaldado por volúmenes locales persistentes.
  • Backend: Microservicio HTTP ligero corriendo en lenguaje Python bajo el puerto 5000.

Comandos principales para la construcción e inicio del entorno:

# Construir las imágenes personalizadas y levantar los contenedores en segundo plano
docker-compose up --build -d

# Verificar el estado operativo de los servicios levantados
docker ps
[Aquí debes incluir tus capturas: Archivos Dockerfile, docker-compose.yml y Pantallazos de acceso desde el navegador al Frontend y Backend]

Componente 3: Orquestación con Kubernetes (20%)

Se desplegó una solución robusta y escalable utilizando Minikube local en la VM principal. Se diseñaron manifiestos declarativos YAML para separar las cargas de trabajo (Deployments) de los accesos a la red (Services).

  • Despliegue: Imagen oficial del servidor Nginx instanciada inicialmente con un total de 2 réplicas estables.
  • Acceso Externo: Exposición de puertos del servicio a través de la directiva NodePort.
  • Escalabilidad Dinámica: Ampliación en caliente del clúster local elevándolo controladamente a 3 réplicas en tiempo real.

Comandos fundamentales aplicados en el clúster:

# Aplicar los manifiestos de despliegue y servicio
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

# Realizar el escalado manual a 3 réplicas operativas
kubectl scale deployment nginx-deployment --replicas=3

# Listar los Pods activos y servicios expuestos
kubectl get pods
kubectl get svc

Conclusiones

Al concluir las fases de integración y pruebas del sistema operativo, el equipo destaca las siguientes deducciones clave:

Eje del Proyecto Conclusión e Impacto Técnico
Virtualización e Interconexión El particionamiento manual y el aislamiento de redes internas simulan fielmente un entorno real de centro de datos corporativo, validando la seguridad del protocolo de comunicación SSH en la capa administrativa.
Portabilidad con Contenedores El uso coordinado de Docker Engine y Docker Compose basado en imágenes limpias como Debian demostró que desacoplar el código de la infraestructura permite desplegar servicios heterogéneos (Nginx/Python) en segundos y de manera repetible.
Alta Disponibilidad e Integridad La orquestación bajo Kubernetes (Minikube) y la capacidad de escalar réplicas en caliente aseguran la resiliencia de la plataforma ante picos inesperados de demanda, sentando las bases esenciales de la ingeniería Cloud moderna.
